Session 2

Question 1: Unification

Consider the logic program containing the facts: 

vertical(point(X,Y),point(X,Y1)).
horizontal(point(X,Y),point(X1,Y)).

Predict the answers prolog will give for the following queries: 

?- vertical(point(1,1),point(1,2)).
?- vertical(point(1,1),point(2,4)).
?- horizontal(point(1,1),point(2,X)).
?- vertical(point(2,3),P).
?- horizontal(point(1,2),point(X,_)).
?- horizontal(point(2,3),point(5,3,1)).

Question 2: Caching

Sometimes it can be useful to add facts and/or rules to the program at runtime, e.g. to prevent deriving things twice. For this purpose Prolog has the following predicates 

assert(+Term): add the clause Term at the end of the prolog database
asserta(+Term): add the clause Term at the beginning of the prolog database

Note, when you assert a predicate that is already defined you get an error. The reason herefore is that the standard interpreter does not allow you to modify predicates at runtime that are already defined. But, using the following constraint-rule in you program fixes the problem: 

:-dynamic xxxxxxx/#.

The explanation of dynamic in the SWI-prolog manual is: 

dynamic +Functor/+Arity, ...: Informs the interpreter that the
definition of the predicate(s) may change during execution
(using assert/1 and/or retract/1).

Consider the following logic program: 

a:-assert((a:-b)),fail.
b:-asserta((b:-c)),fail.
b:-write('Dit is b').
c:-write('Dit is c').

Predict the complete output when we ask the following queries in order: 

?- listing.
?- a.
?- listing.
?- a.
?- listing.
?- a.
?- listing.
?- a.
?- listing.

In the same way we can remove facts and/or rules from the program during runtime. Therefore we can use the following predicate: 

retract(+Term): unify "Term" with the first unifying clause
in the database and remove that clause from the database

Question 3: Lists

In prolog [X|Y] represents a list, where X is the head and Y is the tail of the list. The empty list is represented by []. Obviously, prolog lists have a great analogy with scheme lists in cons-notation.

Translate the following scheme cons-lists to prolog lists: 

(cons a '())
(cons a (cons b (cons c '())))
(cons a L)
(cons a (cons b (cons c L)))
(cons (cons a (cons b (cons c '()))) (cons b (cons d (cons e '()))))

In scheme the construction (cons a (cons b '()) can be simplified by using (list a b). In the same way we can simplify the prolog notation by using [a, b] instead of [a|[b|[]]]. Note that when the tail is a variable we still use the [Head|Tail] notation, e.g. [x,y|Z].

Rewrite the previous cons-lists using this new notation.

To get used to working with lists, write clauses for the following operations on lists: 

my_member(Element,List): check if element belongs to list
my_append(List1,List2,Result): append two lists
rotateleft(List,Result): shift list to the left side
rotateright(List,Result): shift list to the right side

Question 4: Tracing/Debugging

Check the sections about tracing and debugging in the SWI-Prolog manual as they will be usefull to find mistakes in your programs/project.

Consider the following logic program: 

h([],[]).
h([X|Y],Z) :- h(Y,L), append(L,[X],Z).

        1) What does the program?
        2) Give a complete trace of the query h([1,2,3],L).
        3) Predict the outcome of h(L,[1,2,3]).

Look at the three implementations of the sublist relation below. The first one, sub1, is a procedural definition, while the two others are written in a more declarative style. 

sub1(List,Sublist):-
        prefix(List,Sublist).
sub1([_|Tail],Sublist):-
        sub1(Tail,Sublist).

prefix(_,[]).
prefix([X|List1],[X|List2]):-
      	prefix(List1,List2).

sub2(List,Sublist):-
      	append(List1,List2,List),
      	append(List3,Sublist,List1).

sub3(List,Sublist):-
      	append(List1,List2,List),
      	append(Sublist,_,List2).

Use the tracer/debugger to study how efficient the three predicates are. Two predicates almost have the same efficiency. Which ones? Why is the third one less efficient?

Question 5: More on Lists

Write a predicate that computes the reverse of a list, this by using an accumulator, meaning you have to start with 

my_reverse(L,R):-my_reverse(L,[],R).

Write clauses for the following operations on lists:

  • my_last(Element,List): give the last element of a list
  • my_delete(List,Element,Result): delete every occurence of Element in List
  • tailsearch(Element,List,Result): search Element in List and return the sublist containing the Element
  • permutation(List,Permutation): give all possible permutations of a list (HINT: define a permutation on n elements recursively by using a permutation on n-1 elements)
  • my_subset(Set,Sublist): consider Set and Sublist as lists representing sets, and check the subset-relation; be sure your clauses can be used to generate all possible subsets of a given set.
  • powerset(Set,Subsets): compute all subsets (represented by lists) of a given set.
