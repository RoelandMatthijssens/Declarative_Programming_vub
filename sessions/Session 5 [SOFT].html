Session 5

Question 1: Operators of your own

We can define new operators in prolog ourselves by means of a special kind of predicates, called directives, which behave like operator definitions. Such a definition must be given before the first use of the operator in an expression. The syntax to add an operator to your program is (also look in the SWI-prolog manual): 

:- op(precedence,type,name).

Note that no action or operation is defined in the definition of an operator. In principle, no operations on data are associated with an operator. Normally, operators are used as functors to combine objects to structures and not to start actions on data, despite the suggestion of action by the word operator.

The name of an operator must be an atom and the precedence must be a number between 1 en 1200, where a lower priority has a stronger binding. For the type of operator we have three groups to chose from: 

1) infix
	xfx	xfy	yfx
2) prefix
	fx	fy
3) postfix
	xf	yf

where f represents the operator and x and y the arguments. The difference between x and y is associativity, i.e. if arguments with the same precedence are accepted or not. When an argument is denoted using x, there is no associativity for that argument and thus x must be an operator with a strict lower priority. When y is used, left or right associativity is used, depending if y appears left or right from the operator.

Some examples to illustrate: 

xfx means not associative, thus X op Y op Z is not legal and
must be written as either (X op Y) op Z or X op (Y op Z).

xfy means right associative, thus X op Y op Z is accepted and
should be read as op(X, op(Y,Z)).

yfx means left associative, thus X op Y op Z is accepted and
should be read as op(op(X,Y),Z).

Answer the following questions: (you can use the built-in “write_canonical(Term)” predicate to help you examine the structure of a term)

1) Consider the following definitions: 

:- op(300,xfx,plays).
:- op(200,xfy,and).

How does prolog represents the following syntactically correct objects internally: 

tommy plays football and squash
susan plays tennis and basketball and volleyball

2) Give good definitions for the operators “was”, “of” and “the” such that rules of the form 

lydie was the secretary of the department

are accepted by prolog. Answer now the following query: 

?- Who was the secretary of the department.

?- lydie was What.

3) Uses operators so you can remove elements from a list in the following way 

deleting Element from List gives NewList

4) Define the operators “if”, “then”, “else” and ”:=” such that the following is accepted by prolog as a correct term: 

if X > Y then Z := X else Z := Y

Chose your precedence such that “if” is the mainfunctor. Now define the “if” relation as a sort of interpreter for the if-then-else relation of the form 

if Val1 > Val2 then Var := Val3 else Var := Val4

where Val1, Val2, Val3 and Val4 are numbers (or variables that instantiate to numbers) and Var is a variable. The semantics for this relation is: 

if the value of Val1 is bigger then Val2, then Var must be
instantiated with Val3, otherwise with Val4.

An example run of this interpreter is: 

?- X=2, Y=3, Val2 is 2*X, Val4 is 4*X,
   if Y > Val2 then Z := Y else Z := Val4,
   if Z > 5 then W := 1 else W := 0.
X = 2
Y = 3
Z = 8
W = 1
Val2 = 4
Val4 = 8

Question 2: Predicates for input and output

Prolog has a bunch of built-in predicates that work with files and input/output of data. Check carefully the SWI-prolog manual to learn about them.

Solve the following questions:

1) Write a predicate square that asks a number, computes the square and shows the result to the user. Your program must go on and on by asking a number and computing the square until the user gives STOP as input.

2) Implement a predicate writelist(L) that prints a list in an elegant way, e.g. 

?- writelist([a,b,c,d,e,f]).
a b c d e f

3) Write a predicate bars(L), where L is a list of numbers, such that 

?- bars([3,4,6,5]).
***
****
******
*****

4) Suppose you already opened a file containing terms. Implement a predicate showfile(…) which reads the terms from the file and prints them out on the screen together with their ranking number.

5) Suppose you already opened a file containing terms. Write a predicate findallterms(Term) which prints out every term from the file that unifies with Term. Hint: Prevent instantiating Term during the process, because this will prevent that it can be unified with terms occuring later in the file.

6) Write a predicate squeeze that reads a sentence from the input stream and prints out that sentence to the standard output stream in such a way that occurences of more than one space between two words are replaced by one single occurence. Hint, the ascii code for a space is 32, for a dot it is 46.
