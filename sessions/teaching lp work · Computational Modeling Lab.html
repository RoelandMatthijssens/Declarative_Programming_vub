Basis SWI-Prolog

  • To start SWI-prolog type pl on the commandline.
  • Starting up/Loading a file:
  ?- consult(+file).
  + file is the name of a prolog source file that has to be read.
  + file has either no extension or the extension '.pl'
  + file has to be in the current directory
  + never forget to place a point after the query
  + if a file is a list of files, then every file is loaded in order
  + e.g.: the filename is "test.pl", then we type
  ?- consult(test).
  ?- ensure_loaded(+file).
  + is equivalent with consult, but the file is only 
    loaded/consulted if this was not done before.
  • SWI-prolog is left by

?- halt.

  • To get further results of a query type ”;” after a result.
  • To get online help, you have the following predicates at your disposal:
 ?- help(+what).
   + "what" is one of the following:
     # Name/Arity => show a part of the manual about this specific predicate.
     # Name => show a part of the manual about a predicate or 
               interface function with this name
     # e.g.:
     ?- help(append).
 ?- explain(+ToExplain).
   + This predicate explains the given prolog-object "ToExplain", 
   e.g., is an atom, is primitive, ...
   + eg.:
     ?- explain(flight(sn,5016)).
       "flight(sn,5040)" is a compound term
  • You can go through the history of all tried queries using the arrows on your keyboard.
  • You can get a listing of all the code that you have loaded yourself by the command
    ?- listing.
  • If the predicates with the same name are not together in the sourcefile, SWI-prolog will give the following warning: “Clauses of xxxxxxxxxx/# are not together in the sourcefile”. Because warnings can be quite annoying and confusing (like assistants), one can disable them by adding at the top of the source file
    :-style_check(-discontiguous).
    Similarly, they can be back activated:
    :-style_check(+discontiguous).
